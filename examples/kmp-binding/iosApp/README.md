# iOS App

The iOS target uses Kotlin/Native — no JNI is involved.

## Setup

1. Add a cinterop definition that wraps your native C/C++ library headers.
   Create `shared/src/nativeInterop/cinterop/kmmbinding.def`:

   ```
   headers = kmmbinding.h
   headerFilter = kmmbinding/**
   staticLibraries = libkmmbinding.a
   libraryPaths = /path/to/your/lib
   ```

2. Wire it in `shared/build.gradle.kts` under `iosMain`:
   ```kotlin
   iosMain {
       cinterops {
           val kmmbinding by creating {
               defFile("src/nativeInterop/cinterop/kmmbinding.def")
           }
       }
   }
   ```

3. Replace the stub bodies in
   `shared/src/iosMain/kotlin/com/example/kmmbinding/NativeBridge.ios.kt`
   with real calls to the cinterop bindings.

4. Open `iosApp/iosApp.xcodeproj` in Xcode (generated by the KMP Gradle plugin
   after running `./gradlew :shared:linkDebugFrameworkIosSimulatorArm64`).

The JNI binding generator does **not** process iOS sources — it only covers
`androidMain` and `desktopMain`.
